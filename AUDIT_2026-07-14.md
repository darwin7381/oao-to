# 🔍 OAO.TO 全面審計報告（2026-07-14）

> 五路平行審計：登入/認證、權限/角色、兩個後台 UI、核心功能（連結/重定向/KV-D1）、API 平台。
> 審計原則：只報 ≥ 中信心的真實問題，每項標 檔案:行號、嚴重度、**修 or 重寫**。

---

## ⚖️ 總判決

系統**功能面向完整**，但有三類系統性問題，其中多處確實需要**重寫**而非修補：

1. 🔴 **正在洩漏的 live 安全漏洞**（無認證端點、IDOR）— 需立即滅火
2. 🔴 **架構缺陷需重寫**：Rate Limiter（KV race 可完全繞過）、KV↔D1 雙寫無一致性模型、認證 token 流程
3. 🟡 **大量半成品 UI**：多個管理頁面 runtime crash、假功能（Settings 只 setTimeout）、裝飾性按鈕

**需要「重寫」的清單（Joey 的直覺是對的）：**
- Rate Limiter → 改 Durable Objects 或 Cloudflare Rate Limiting binding
- KV↔D1 一致性 → 確立 D1 為 source of truth、KV 降為快取、統一寫入入口
- 認證 token 流程 → httpOnly cookie 取代 URL 明文 token + 401 處理 + 縮短 JWT TTL
- `/shorten` 公開端點 → 廢棄或重寫（現在繞過 credits + auth + rate limit）
- 若干管理頁面（Stats/Settings/ApiKeysMonitoring）→ 型別對齊後端、接真實 API
