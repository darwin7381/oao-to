## 🚨 Hotfix 類型

### 1. 緊急修復 (Emergency Hotfix)
- 影響生產環境的嚴重問題
- 需要立即修復
- 可能跳過正常部署流程

### 2. 問題排查 (Troubleshooting)
- 生產環境問題診斷
- 根本原因分析
- 解決方案記錄

### 3. 變更審查 (Change Review)
- 重大變更的事後審查
- 問題總結
- 改進建議

---

## 📝 記錄規範

### 命名格式

```
HOTFIX_YYYY-MM-DD_描述.md        # 緊急修復
FINAL_FIX_YYYY-MM-DD.md         # 最終修復
CHANGE_REVIEW_YYYY-MM-DD.md     # 變更審查
INCIDENT_YYYY-MM-DD_描述.md     # 事故記錄
```

### 記錄內容應包含

1. **問題描述**
   - 發現時間
   - 問題現象
   - 影響範圍
   - 嚴重程度

2. **根本原因分析**
   - 技術原因
   - 流程問題
   - 環境因素

3. **修復方案**
   - 臨時方案（如有）
   - 最終方案
   - 實施步驟

4. **驗證結果**
   - 修復驗證
   - 功能測試
   - 監控結果

5. **經驗總結**
   - 學到的教訓
   - 改進建議
   - 預防措施

---

## ⚠️ 緊急修復流程

### 發現問題
1. 立即評估影響範圍和嚴重程度
2. 決定是否需要緊急修復
3. 通知相關人員

### 實施修復
1. 快速定位問題根源
2. 實施臨時緩解措施（如需要）
3. 開發並測試修復方案
4. 部署到生產環境

### 事後處理
1. 完整記錄問題和修復過程
2. 創建 Hotfix 記錄文檔
3. 審查並總結經驗
4. 更新相關流程和文檔
